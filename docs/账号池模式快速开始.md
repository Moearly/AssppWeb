# 账号池模式快速开始指南

## 📋 什么是账号池模式？

**账号池模式**是一种共享账号方案，让用户无需自己的 Apple ID 也能下载应用。

### 工作流程

```
管理员提供共享 Apple ID
  ↓
加密存储在服务器数据库
  ↓
用户访问应用商店
  ↓
点击"获取账号"
  ↓
服务器分配一个可用账号到浏览器
  ↓
【关键】浏览器自动登录这个共享账号（libcurl.js）
  ↓
用户正常搜索和下载应用
  ↓
用完后账号自动归还到池中
```

### 核心优势

- ✅ **用户友好**：无需自己的 Apple ID
- ✅ **技术可靠**：浏览器端登录（真实设备模拟）
- ✅ **通过验证**：完全模拟真实 iOS 设备
- ✅ **保持零信任**：服务器仍然不见 Apple 会话内容

### 与零信任模式对比

| 特性 | 零信任模式 | 账号池模式 |
|------|-----------|-----------|
| 用户账号 | 自己的 Apple ID | 共享 Apple ID |
| 登录方式 | 手动输入 | 系统自动分配并登录 |
| 认证位置 | 浏览器（libcurl.js） | 浏览器（libcurl.js）✅ |
| 服务器角色 | 盲 TCP 代理 | 盲 TCP 代理 ✅ |
| 适用场景 | 隐私要求高 | 快速体验 |

## 📱 常见问题

### Q1: 用户需要什么？

**A**: 用户完全不需要自己的 Apple ID！系统会自动分配共享账号到浏览器，用户只需要点击"获取账号"，等待浏览器自动登录即可。

### Q2: 为什么要在浏览器登录？

**A**: 
- 浏览器端认证（libcurl.js + Mbed TLS）可以完美模拟真实 iOS 设备
- 正确的 TLS 指纹，通过所有 Apple 验证
- 服务器仍然是盲 TCP 代理，不见 Apple 会话内容（保持零信任）
- 如果在服务器端直接调用 Apple API，可能被检测为非法客户端

### Q3: 账号安全吗？

**A**: 
- 账号在服务器加密存储（AES-256-GCM）
- 只在需要时临时分配到浏览器
- 用完后自动归还，不会保留在浏览器
- 这些是共享账号，不是个人隐私账号

### Q4: 如何获取应用的 Bundle ID？

**A**: 
- 方法1：在 App Store 网页版搜索应用，查看 URL 中的 ID
- 方法2：使用抓包工具查看应用包名
- 方法3：搜索"应用名称 Bundle ID"

### Q6: 账号添加失败怎么办？

**A**: 检查：
1. 账号密码是否正确
2. 是否需要验证码（开启了2FA）
3. 设备ID是否有效（12位十六进制）
4. 网络连接是否正常

### Q5: 账号显示"已过期"怎么办？

**A**: 
1. 点击账号右侧的"健康检查"按钮
2. 如果提示重新认证，输入验证码
3. 或者删除后重新添加该账号

### Q7: 普通用户如何使用？

**A**: 
1. 访问 https://chatvpn.site/pool/store
2. 点击"获取账号"
3. 等待浏览器自动登录（5-10秒）
4. 搜索应用并下载
5. 就像使用自己的 Apple ID 一样

## 🔐 安全说明

1. **账号加密**：
   - 账号密码使用 AES-256-GCM 加密
   - 加密密钥存储在服务器环境变量中
   - 数据库中只存储加密后的数据

2. **API Key 管理**：
   - Admin API Key 使用 SHA-256 哈希验证
   - 仅存储在浏览器 localStorage 中
   - 不会上传到服务器

3. **账号使用**：
   - 每个账号都有冷却时间（防止频繁使用）
   - 自动健康检查
   - 错误次数过多自动禁用
