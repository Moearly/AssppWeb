## 📋 完整部署配置

### 1. 服务器配置

#### 系统要求
- **操作系统**: Debian/Ubuntu
- **内存**: 最低 1GB
- **磁盘**: 最低 10GB
- **端口**: 80 (HTTP)

#### Docker Compose 配置

**文件**: `/root/AssppWeb/compose.yml`

```yaml
services:
  asspp:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "80:8080"  # 重要：使用端口 80
    volumes:
      - ./mnt/asspp-data:/data
    environment:
      - DATA_DIR=/data
      - PORT=8080
      - ACCOUNT_POOL_KEY=${ACCOUNT_POOL_KEY}
      - ADMIN_API_KEY=${ADMIN_API_KEY}
      - UNSAFE_DANGEROUSLY_DISABLE_HTTPS_REDIRECT=true
    restart: unless-stopped
    logging:
      driver: "json-file"
      options:
        max-size: "128m"
        max-file: "5"
```

### 2. Cloudflare 配置

#### DNS 配置

**步骤**:

1. 登录 Cloudflare Dashboard: https://dash.cloudflare.com
2. 选择域名 `chatvpn.site`
3. 进入 **DNS** → **记录**
4. 添加 A 记录：
   ```
   类型: A
   名称: @ (根域名)
   IPv4地址: 156.226.173.202
   代理状态: ✅ 已代理（橙色云朵图标）
   TTL: 自动
   ```

**API 配置命令**:
```bash
# Zone ID
ZONE_ID="b44740042fdfcf5b3e60e8655745571a"

# 创建 DNS A 记录（已代理）
curl -X POST "https://api.cloudflare.com/client/v4/zones/$ZONE_ID/dns_records" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  --data '{
    "type": "A",
    "name": "@",
    "content": "156.226.173.202",
    "ttl": 1,
    "proxied": true
  }'
```

#### SSL/TLS 配置

**步骤**:

1. 进入 **SSL/TLS** → **概述**
2. 选择 **灵活（Flexible）** 模式

**API 配置命令**:
```bash
curl -X PATCH "https://api.cloudflare.com/client/v4/zones/$ZONE_ID/settings/ssl" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  --data '{"value": "flexible"}'
```

**Flexible SSL 模式说明**:
```
用户浏览器 <--HTTPS (加密)--> Cloudflare <--HTTP (明文)--> 源服务器
```

- ✅ 用户到 Cloudflare：HTTPS 加密（免费 SSL 证书）
- ✅ Cloudflare 到源服务器：HTTP（端口 80）
- ✅ 无需在源服务器安装 SSL 证书

## 🚧 遇到的问题和解决方案

### 问题 1: Cloudflare HTTPS 返回 521 错误

**现象**:
```bash
$ curl -I https://chatvpn.site
HTTP/2 521
```

**错误原因**:
1. ❌ 服务器端口配置错误（使用了 8080 而非 80）
2. ❌ DNS 传播时间不足
3. ❌ Cloudflare SSL 证书未完全激活

**解决方案**:
```bash
# 1. 修改 Docker 端口映射为 80
# compose.yml: "80:8080" (主机80端口映射到容器8080端口)

# 2. 确认 SSL 模式为 Flexible
curl -X PATCH "https://api.cloudflare.com/client/v4/zones/$ZONE_ID/settings/ssl" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"value": "flexible"}'

# 3. 启用 Cloudflare 代理
curl -X PATCH "https://api.cloudflare.com/client/v4/zones/$ZONE_ID/dns_records/$RECORD_ID" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"proxied": true}'

# 4. 等待 20-30 秒让配置生效
sleep 30
```

**关键经验**: 
- ⏱️ **必须等待 20-30 秒**让 Cloudflare SSL 证书和边缘节点完全生效
- 🔧 **端口必须是 80**（标准 HTTP 端口），Cloudflare 会自动处理 HTTPS

---

### 问题 2: 环境变量未生效

**现象**:
```bash
# HTTPS 重定向导致循环
HTTP/1.1 301 Moved Permanently
Location: https://...
```

**错误原因**:
服务器端启用了 HTTPS 重定向，但没有 SSL 证书，导致与 Cloudflare Flexible 模式冲突。

**解决方案**:

在 `.env.generated` 中添加：
```bash
UNSAFE_DANGEROUSLY_DISABLE_HTTPS_REDIRECT=true
```

在 `compose.yml` 中传递环境变量：
```yaml
environment:
  - UNSAFE_DANGEROUSLY_DISABLE_HTTPS_REDIRECT=${UNSAFE_DANGEROUSLY_DISABLE_HTTPS_REDIRECT}
```

重启容器：
```bash
cd /root/AssppWeb
docker compose down
docker compose up -d
```

---

### 问题 3: 容器内路径错误

**现象**:
```bash
Error: No such container:path: assppweb-asspp-1:/app/frontend/dist
```

**错误原因**:
项目使用多阶段构建，前端构建产物在 `/app/public`，不是 `/app/frontend/dist`。

**解决方案**:

查看 Dockerfile 确认正确路径：
```dockerfile
# Stage 3: Runtime
COPY --from=frontend-build /app/frontend/dist ./public
```

更新前端时需要重新构建整个镜像：
```bash
cd /root/AssppWeb
docker compose up -d --build
```

## 🎓 核心经验总结

### 1. Cloudflare Flexible SSL 配置要点

✅ **正确配置**:
- 源服务器监听端口 80（HTTP）
- Cloudflare SSL 模式设为 Flexible
- DNS 记录启用代理（橙色云朵）
- 服务器禁用 HTTPS 重定向
- **等待 20-30 秒让配置生效**

❌ **常见错误**:
- 使用 8080 等非标准端口
- 源服务器配置了 HTTPS 重定向
- 没有等待足够时间
- SSL 模式设置错误

### 2. Docker 端口映射

**正确格式**: `"主机端口:容器端口"`

```yaml
ports:
  - "80:8080"  # 主机80映射到容器8080
```

**不要使用**:
- `"8080:8080"` - Cloudflare Flexible 需要端口 80
- `"443:8080"` - 服务器无 SSL 证书

### 3. 环境变量传递

**必须两处配置**:

1. `.env.generated` 文件：
   ```bash
   UNSAFE_DANGEROUSLY_DISABLE_HTTPS_REDIRECT=true
   ```

2. `compose.yml` 传递：
   ```yaml
   environment:
     - UNSAFE_DANGEROUSLY_DISABLE_HTTPS_REDIRECT=${UNSAFE_DANGEROUSLY_DISABLE_HTTPS_REDIRECT}
   ```

### 4. 等待时间的重要性

**Cloudflare 配置生效时间**:
- DNS 更改: 10-30 秒
- SSL 证书激活: 20-60 秒
- 边缘节点同步: 10-30 秒

**建议**: 配置后至少等待 30 秒再测试
