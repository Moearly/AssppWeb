# 项目概述

## AssppWeb 是什么？

AssppWeb 是一个基于 Web 的 iOS 应用获取工具，允许用户通过浏览器使用 Apple ID 下载 App Store 上架应用的 IPA 文件。

### 核心特性

- ✅ **零信任架构**: 服务器永远无法看到用户的 Apple 凭据
- ✅ **浏览器端加密**: 所有 Apple API 通信在浏览器内通过 WebAssembly 完成 TLS 1.3 加密
- ✅ **盲代理模式**: 服务器仅作为 TCP 中继，无法解密流量
- ✅ **多账号管理**: 支持多个 Apple ID，每个账号独立的设备标识符和 Pod 路由
- ✅ **版本历史**: 支持下载应用的历史版本（账号购买过的应用）
- ✅ **自动编译**: 自动下载 IPA、注入 SINF 签名、生成可安装包

### 技术栈

#### 前端
- **React 19** - UI 框架
- **TypeScript** - 类型安全
- **Vite** - 构建工具
- **Tailwind CSS 4** - 样式框架
- **Zustand** - 状态管理
- **IndexedDB (idb)** - 浏览器本地存储
- **libcurl.js (WASM)** - TLS 1.3 客户端

#### 后端
- **Node.js + Express** - HTTP 服务器
- **TypeScript (ESM)** - 服务端逻辑
- **@mercuryworkshop/wisp-js** - Wisp 协议代理
- **Axios** - HTTP 客户端
- **ADM-ZIP** - IPA 文件处理
- **SQLite (可选)** - 账号池数据库

#### 部署
- **Docker + Docker Compose** - 容器化部署
- **Caddy (可选)** - HTTPS 反向代理
- **Cloudflare (可选)** - CDN 加速

# 零信任架构原理

## 传统方案的问题

传统的 iOS 应用下载工具通常要求用户将 Apple ID 和密码发送到服务器，由服务器代理完成所有操作：

```
❌ 传统架构（不安全）

[浏览器] → 发送明文凭据 → [服务器]
                               ↓
                        存储凭据/直接使用
                               ↓
                        连接 Apple 服务器
```

**风险**:
- 服务器管理员可以窃取用户凭据
- 数据库泄露导致批量账号被盗
- 中间人攻击可能截获传输中的凭据

## AssppWeb 的零信任方案

AssppWeb 通过 **端到端加密** + **盲代理** 的设计彻底解决了这个问题：

```
✅ 零信任架构（安全）

[浏览器 (libcurl.js WASM)]
    ↓ 建立 TLS 1.3 加密连接
    ↓ (密钥仅存在于浏览器内存)
    ↓
[Wisp 协议 over WebSocket]
    ↓ 加密流量（服务器无法解密）
    ↓
[服务器 (Wisp 盲代理)]
    ↓ 仅转发 TCP 字节流
    ↓
[Apple 服务器]
```

### 核心安全机制

#### 1. 凭据永不离开浏览器

```typescript
// 前端：凭据存储在 IndexedDB
import { openDB } from 'idb';

const db = await openDB('asspp', 1, {
  upgrade(db) {
    db.createObjectStore('accounts');
  }
});

// 存储账号（仅在浏览器本地）
await db.put('accounts', {
  email: 'user@example.com',
  password: 'secret',        // ⚠️ 仅存在于用户浏览器
  passwordToken: 'token',    // ⚠️ 从不发送到服务器
  cookies: {...}
}, accountHash);
```

**安全特性**:
- IndexedDB 受浏览器同源策略保护
- 服务器无法访问这些数据
- 数据仅存在于用户设备

#### 2. 浏览器内完成 TLS 加密

```typescript
// libcurl.js 在浏览器内建立 TLS 连接
import { libcurl } from 'libcurl.js-omni';

const response = await libcurl.fetch('https://buy.itunes.apple.com/...', {
  method: 'POST',
  headers: {
    'X-Dsid': account.DSID,
    'Cookie': account.cookies,
    'X-Apple-I-MD': passwordToken  // ⚠️ 在 TLS 加密层内
  },
  body: plistData,
  _libcurl_wisp_proxy: 'ws://server.com/wisp/'  // 通过盲代理
});
```

**关键点**:
- TLS 握手在浏览器 WASM 中完成
- 服务器只能看到加密后的字节流
- Apple 凭据在 TLS 加密层保护下传输

#### 3. 服务器是盲代理

```typescript
// backend/src/services/wsProxy.ts
import { WispServer } from '@mercuryworkshop/wisp-js';

const wispServer = new WispServer({
  hostname_whitelist: [
    'buy.itunes.apple.com',
    'auth.itunes.apple.com',
    /^p\d+-buy\.itunes\.apple\.com$/  // Pod-based hosts
  ],
  allow_direct_ip: false,     // 禁止 IP 直连
  allow_loopback_ips: false   // 禁止回环地址
});

// 服务器只做字节流转发
wispServer.on('stream', (stream, host, port) => {
  const socket = net.connect(port, host);
  
  // 双向盲转发
  stream.pipe(socket);
  socket.pipe(stream);
  
  // ⚠️ 服务器无法解密 TLS 流量
});
```

**安全保证**:
- 服务器只验证目标主机白名单
- 不解析、不记录、不缓存流量内容
- 即使服务器被攻破，也无法获取用户凭据

# 核心技术详解

## libcurl.js + Mbed TLS

### WebAssembly 架构

```
┌─────────────────────────────────────────────┐
│           浏览器 JavaScript                  │
│  ┌───────────────────────────────────────┐  │
│  │  libcurl.js API                       │  │
│  │  ↓                                    │  │
│  │  WebAssembly (libcurl.wasm)           │  │
│  │  ├─ libcurl (HTTP 客户端)             │  │
│  │  └─ Mbed TLS (TLS 1.3 实现)           │  │
│  │     ↓                                 │  │
│  │  Wisp 协议封装                         │  │
│  └───────────────────────────────────────┘  │
│              ↓ WebSocket                    │
└──────────────┼──────────────────────────────┘
               ↓
┌──────────────┼──────────────────────────────┐
│              ↓ TCP 中继                     │
│       Wisp 盲代理服务器                      │
│              ↓                              │
└──────────────┼──────────────────────────────┘
               ↓ TLS 1.3 (加密)
┌──────────────┼──────────────────────────────┐
│       Apple 服务器                           │
└─────────────────────────────────────────────┘
```

### TLS 1.3 握手

```
浏览器 (libcurl.js)                     Apple 服务器
      │                                      │
      │──── ClientHello ───────────────────>│
      │     (TLS 1.3, 支持的加密套件)         │
      │                                      │
      │<─── ServerHello ────────────────────│
      │     (选择加密套件, 服务器证书)         │
      │                                      │
      │──── [加密] Finished ───────────────>│
      │                                      │
      │<─── [加密] Finished ────────────────│
      │                                      │
      │<══════ 加密通道建立 ═══════════════>│
      │                                      │
      │──── [加密] HTTP Request ──────────>│
      │     POST /WebObjects/MZFinance...   │
      │                                      │
      │<─── [加密] HTTP Response ───────────│
      │     (Apple plist 响应)               │
```

**关键点**:
- TLS 握手和会话密钥生成完全在浏览器 WASM 内存中完成
- 服务器只看到加密的字节流，无法解密
- 即使服务器记录所有流量，也无法破解 TLS 1.3 加密

## Wisp 协议

### 多路复用

Wisp 协议允许在单个 WebSocket 连接上承载多个 TCP 流：

```typescript
// Wisp 数据包格式
interface WispPacket {
  streamId: number;      // 流标识符 (TCP 连接 ID)
  type: 'CONNECT' | 'DATA' | 'CLOSE';
  payload: Uint8Array;
}

// 浏览器端
const ws = new WebSocket('wss://server.com/wisp/');

// 1. 请求建立 TCP 连接到 Apple 服务器
ws.send({
  type: 'CONNECT',
  streamId: 1,
  payload: { host: 'buy.itunes.apple.com', port: 443 }
});

// 2. 发送加密的 HTTP 请求
ws.send({
  type: 'DATA',
  streamId: 1,
  payload: tlsEncryptedBytes
});

// 3. 接收加密的响应
ws.onmessage = (event) => {
  const packet = parseWispPacket(event.data);
  if (packet.type === 'DATA') {
    libcurl.processResponse(packet.payload);
  }
};
```

### 主机白名单

```typescript
// backend/src/services/wsProxy.ts
const wispServer = new WispServer({
  hostname_whitelist: [
    'auth.itunes.apple.com',        // 认证端点
    'buy.itunes.apple.com',         // 购买端点
    'init.itunes.apple.com',        // Bag 端点
    /^p\d+-buy\.itunes\.apple\.com$/ // Pod-based hosts (p25-buy, p42-buy)
  ],
  port_whitelist: [443],            // 仅允许 HTTPS
  allow_direct_ip: false,           // 禁止 IP 直连
  allow_loopback_ips: false,        // 禁止回环
  allow_private_ips: true           // 允许私有 IP（Docker 内部）
});
```

**安全考虑**:
- 白名单防止代理被滥用
- 仅允许访问 Apple 官方域名
- 禁止访问内网和敏感服务

### Account Hash 设计

```typescript
// 生成 accountHash (SHA-256)
export async function accountHash(email: string): Promise<string> {
  const encoder = new TextEncoder();
  const data = encoder.encode(email);
  const hash = await crypto.subtle.digest('SHA-256', data);
  return Array.from(new Uint8Array(hash))
    .map(b => b.toString(16).padStart(2, '0'))
    .join('');
}

// 用途：
// 1. IndexedDB 主键（浏览器本地）
// 2. 下载任务关联（服务器端）
// 3. 公开标识符（非敏感数据）
```

**为什么使用 SHA-256？**
- 单向哈希，无法反推 email
- 固定长度（64 字符）
- 作为公开标识符安全

# Apple 协议实现

## 1. Bag Endpoint 解析

Bag 端点返回 Apple 服务的配置信息：

```typescript
// frontend/src/apple/bag.ts
export async function fetchBag(deviceId: string): Promise<AppleBag> {
  // 通过后端代理获取 bag.xml
  const response = await axios.get(`/api/bag?guid=${deviceId}`);
  const xml = response.data;
  
  // 解析 XML
  const parser = new DOMParser();
  const doc = parser.parseFromString(xml, 'text/xml');
  
  // 提取认证端点
  const authNode = doc.querySelector('key:contains("authenticateAccount") + string');
  const authEndpoint = authNode?.textContent || 'https://buy.itunes.apple.com/...';
  
  return { authenticateAccount: authEndpoint, /* ... */ };
}
```

**后端代理**:

```typescript
// backend/src/routes/bag.ts
app.get('/api/bag', async (req, res) => {
  const { guid } = req.query;
  
  const response = await axios.get('https://init.itunes.apple.com/bag.xml', {
    params: { guid },
    headers: {
      'User-Agent': 'Configurator/2.15',
      'Accept': 'application/xml'
    },
    timeout: 10000
  });
  
  res.set('Content-Type', 'application/xml');
  res.send(response.data);
});
```

**为什么需要后端代理？**
- CORS 限制：浏览器无法直接跨域请求 Apple 服务器
- Bag 数据是公开的，不包含敏感信息
- 后端代理不涉及用户凭据

## 2. 认证流程

```typescript
// frontend/src/apple/authenticate.ts
export async function authenticate(
  email: string,
  password: string,
  deviceId: string,
  country: string
): Promise<Account> {
  // 1. 获取 Bag 端点
  const bag = await fetchBag(deviceId);
  
  // 2. 构建认证 plist
  const authPlist = buildPlist({
    appleId: email,
    password: password,
    attempt: 4,
    createSession: true,
    guid: deviceId,
    rmp: '0',
    why: 'signIn'
  });
  
  // 3. 通过 libcurl.js 发送请求（TLS 1.3 加密）
  const response = await libcurl.fetch(bag.authenticateAccount, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/x-apple-plist',
      'User-Agent': 'Configurator/2.15',
      'X-Apple-I-Client-Time': new Date().toISOString(),
      'X-Apple-I-TimeZone': 'UTC'
    },
    body: authPlist,
    _libcurl_wisp_proxy: '/wisp/',  // 使用 Wisp 代理
    _libcurl_http_version: 1.1       // 强制 HTTP/1.1
  });
  
  // 4. 解析响应
  const result = parsePlist(await response.text());
  
  if (result.failureType) {
    throw new Error(`Authentication failed: ${result.customerMessage}`);
  }
  
  // 5. 提取凭据
  return {
    email,
    password,
    passwordToken: result.passwordToken,
    DSID: result.dsPersonId,
    cookies: extractCookies(response.headers),
    country,
    deviceIdentifier: deviceId,
    pod: extractPod(response.headers)  // 从 Set-Cookie 提取 pod
  };
}
```

## 3. 购买许可

```typescript
// frontend/src/apple/purchase.ts
export async function purchaseApp(
  account: Account,
  appId: number,
  country: string
): Promise<Sinf[]> {
  // 1. 构建购买 plist
  const purchasePlist = buildPlist({
    creditDisplay: '',
    guid: account.deviceIdentifier,
    salableAdamId: appId,
    appExtVrsId: 0  // 0 = latest version
  });
  
  // 2. 发送购买请求
  const response = await libcurl.fetch(
    `https://${purchaseAPIHost(account.pod)}/WebObjects/MZFinance.woa/wa/volumeStoreDownloadProduct`,
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/x-apple-plist',
        'iCloud-DSID': account.DSID,
        'X-Dsid': account.DSID,
        'X-Apple-I-MD-M': account.passwordToken,
        'Cookie': formatCookies(account.cookies)
      },
      body: purchasePlist,
      _libcurl_wisp_proxy: '/wisp/'
    }
  );
  
  // 3. 解析 SINF（DRM 签名）
  const result = parsePlist(await response.text());
  
  if (!result.songList || result.songList.length === 0) {
    throw new Error('Failed to acquire license');
  }
  
  return result.songList.map((song: any) => ({
    id: song.metadata.itemId,
    sinf: song.sinf  // Base64-encoded SINF data
  }));
}
```

## 5. IPA 下载与 SINF 注入

这部分在**后端**完成：

```typescript
// backend/src/services/downloadManager.ts
async function downloadAndInjectIPA(task: DownloadTask): Promise<string> {
  // 1. 从 Apple CDN 下载 IPA
  const ipaPath = path.join(DOWNLOAD_DIR, `${task.id}.ipa`);
  await downloadFile(task.downloadURL, ipaPath);
  
  // 2. 注入 SINF
  const zip = new AdmZip(ipaPath);
  const entries = zip.getEntries();
  
  // 找到 .app 目录
  const appEntry = entries.find(e => e.entryName.endsWith('.app/'));
  const appDir = appEntry.entryName;
  
  // 注入 SINF 文件到 SC_Info/ 目录
  task.sinfs.forEach((sinf, index) => {
    zip.addFile(
      `${appDir}SC_Info/Manifest${index}.sinf`,
      Buffer.from(sinf, 'base64')
    );
  });
  
  // 3. 注入 iTunesMetadata.plist（可选）
  if (task.metadata) {
    zip.addFile(
      'iTunesMetadata.plist',
      Buffer.from(task.metadata, 'base64')
    );
  }
  
  // 4. 保存修改后的 IPA
  zip.writeZip(ipaPath);
  
  return ipaPath;
}
```

# 故障排查

## 常见错误

### 1. 认证失败 (2034/2042)

**错误码**：`failureType: 2034` 或 `2042`

**原因**：Token 过期或账号状态改变

**解决**：
```typescript
// 检测 Token 过期并重新认证
if (error.failureType === 2034 || error.failureType === 2042) {
  // 删除旧账号
  await db.delete('accounts', accountHash);
  
  // 提示用户重新添加账号
  alert('Account session expired. Please re-add your account.');
}
```

### 2. Wisp 连接失败

**错误**：`WebSocket connection failed`

**原因**：
- 服务器未运行
- 防火墙阻止 WebSocket
- Wisp 路径错误

**解决**：
```typescript
// 测试 Wisp 连接
const ws = new WebSocket('ws://localhost:8080/wisp/');
ws.onopen = () => console.log('Wisp connected');
ws.onerror = (e) => console.error('Wisp failed:', e);
```

### 3. SINF 注入失败

**错误**：IPA 安装后闪退

**原因**：SINF 数据损坏或缺失

**解决**：
```bash
# 检查 IPA 结构
unzip -l app.ipa | grep SC_Info

# 应该看到：
# Payload/App.app/SC_Info/Manifest0.sinf
# Payload/App.app/SC_Info/Manifest1.sinf
```

AssppWeb 的核心优势：

1. **零信任架构**：服务器永远看不到用户凭据
2. **浏览器端加密**：所有敏感操作在浏览器内完成
3. **盲代理模式**：服务器仅做 TCP 中继
4. **官方 API**：使用 Apple 公开的协议
5. **开源透明**：所有代码可审计

这些设计确保了 AssppWeb 在提供便利的同时，最大程度保护用户隐私和账号安全。
