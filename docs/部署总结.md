## 📋 完整的部署流程

### 服务器端部署

#### 方式一: 自动部署（推荐）

```bash
# 1. 克隆代码
git clone https://github.com/your-repo/AssppWeb.git
cd AssppWeb

# 2. 一键部署（自动生成密钥）
bash deploy-manual.sh

# 3. 记录 API Key
# 脚本会输出 ADMIN_API_KEY，请妥善保存
```

#### 方式二: 手动部署

```bash
# 1. 生成密钥
export ADMIN_API_KEY=$(openssl rand -hex 32)
export ACCOUNT_POOL_KEY=$(openssl rand -hex 32)

# 2. 保存配置
echo "ADMIN_API_KEY=$ADMIN_API_KEY" > .env.generated
echo "ACCOUNT_POOL_KEY=$ACCOUNT_POOL_KEY" >> .env.generated

# 3. 构建并启动
docker compose build
docker compose up -d

# 4. 查看日志
docker compose logs -f
```

## 🔐 安全配置

### 1. 生产环境建议

```bash
# 使用 HTTPS（通过 Nginx/Caddy）
server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 2. 防火墙设置

```bash
# 仅允许必要端口
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP
ufw allow 443/tcp   # HTTPS
ufw enable

# 限制管理端口（如需要）
ufw allow from <admin-ip> to any port 8080
```

### 3. API Key 轮换

```bash
# 每3个月轮换一次
export NEW_API_KEY=$(openssl rand -hex 32)
echo "ADMIN_API_KEY=$NEW_API_KEY" > .env.generated

# 重启服务
docker compose restart

# 通知所有管理员更新 API Key
```

### 数据备份

```bash
# 备份脚本（建议 cron 定时执行）
#!/bin/bash
BACKUP_DIR="/backup/assppweb"
DATE=$(date +%Y%m%d-%H%M%S)

mkdir -p "$BACKUP_DIR"

# 备份数据库
docker cp assppweb-asspp-1:/app/data/pool.db "$BACKUP_DIR/pool-$DATE.db"

# 备份配置
cp .env.generated "$BACKUP_DIR/env-$DATE.txt"

# 打包
tar -czf "$BACKUP_DIR/backup-$DATE.tar.gz" \
  "$BACKUP_DIR/pool-$DATE.db" \
  "$BACKUP_DIR/env-$DATE.txt"

# 清理旧备份（保留最近7天）
find "$BACKUP_DIR" -name "backup-*.tar.gz" -mtime +7 -delete
```

## ❓ 常见问题

### Q1: 忘记 API Key 怎么办？

**解决方案**:
```bash
# 查看配置文件
cat .env.generated | grep ADMIN_API_KEY

# 或重新生成
bash deploy-manual.sh  # 会提示是否覆盖
```

### Q2: 用户看不到应用列表？

**原因**: 白名单为空

**解决方案**:
```bash
# 方式一: Web 界面添加
访问 "管理" → "白名单管理" → "搜索应用"

# 方式二: CLI 工具添加
bash admin-cli.sh whitelist:add
```

### Q3: 账号添加失败 "invalid credentials"？

**原因**: Apple ID 密码错误或需要 2FA

**解决方案**:
1. 先在真实 iPhone 上登录该 Apple ID
2. 确保账号可用且无安全警告
3. 如有 2FA，准备好验证码

### Q4: 如何迁移到新服务器？

**步骤**:
```bash
# 旧服务器
tar -czf backup.tar.gz data/ .env.generated
scp backup.tar.gz new-server:/root/

# 新服务器
cd /root/AssppWeb
tar -xzf ../backup.tar.gz
source .env.generated
docker compose up -d --build
```
