## 🗄️ 数据库结构

### account_pool
```sql
- id: 账号ID
- email: Apple ID 邮箱
- email_hash: 邮箱SHA-256
- password: AES-256-GCM加密
- country: 国家代码
- device_identifier: 设备ID
- status: active|disabled|expired|error
- usage_count: 使用次数
- last_used_at: 最后使用时间
- pod: Apple服务器pod
```

### app_whitelist
```sql
- id: 应用ID
- software_id: iTunes软件ID
- bundle_id: Bundle标识符
- name: 应用名称
- country: 国家代码
- artwork_url: 图标URL
- version: 版本号
- enabled: 0|1
```

### health_log
```sql
- id: 日志ID
- account_id: 关联账号
- status: healthy|token_expired|locked|error
- message: 详细信息
- checked_at: 检查时间
```

## 🔐 安全机制

### 加密存储
- **密码**：AES-256-GCM加密（服务器端）
- **Cookie**：AES-256-GCM加密
- **Token**：AES-256-GCM加密
- **密钥来源**：环境变量 `ACCOUNT_POOL_KEY`

### 访问控制
- **管理员API**：需要 `x-admin-key` 请求头
- **密钥存储**：浏览器localStorage（非传输）
- **权限分离**：管理员/用户功能完全隔离

### 速率限制
- **账号级别**：每个账号有冷却期
- **全局限制**：避免单账号频繁使用
- **健康监控**：自动检测异常账号

## ❓ 常见问题

### Q: 页面空白或显示错误？
A: 清除浏览器缓存后重新访问（见上方"清除浏览器缓存"部分）

### Q: API Key测试连接失败？
A: 检查：
1. API Key是否正确复制（无多余空格）
2. 网络是否正常
3. 服务器是否运行：`docker compose ps`

### Q: 管理员菜单不显示？
A: 
1. 确认已成功配置API Key
2. 检查localStorage：打开DevTools → Application → Local Storage
3. 应该看到 `adminApiKey` 键

### Q: 一键下载失败？
A: 检查：
1. 账号池是否有可用账号
2. 账号状态是否为 `active`
3. 账号国家是否匹配应用国家
4. 查看后端日志：`docker compose logs asspp`
