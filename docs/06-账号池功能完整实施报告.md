## 🔐 安全机制

### 1. 账号池加密

- **算法**: AES-256-GCM
- **密钥**: 从环境变量 `ACCOUNT_POOL_KEY` 读取（256-bit）
- **加密字段**: `password`, `passwordToken`, `cookies`, `DSID`
- **明文字段**: `email`, `country`, `device_identifier`, `status`, `pod`

### 2. 管理员认证

- **方式**: HTTP Header `x-admin-key`
- **密钥**: 从环境变量 `ADMIN_API_KEY` 读取
- **作用域**: 仅管理员 API (`/api/admin/*`)
- **存储**: 浏览器 localStorage（仅客户端）

### 3. 账号选择策略

- **负载均衡**: 优先选择使用次数少的账号
- **冷却机制**: 同一账号两次使用间隔至少 5 分钟
- **限流保护**: 每个账号每小时最多 10 次下载
- **健康检查**: 自动跳过已过期/锁定的账号

### 4. 零信任原则保留

- **客户端加密**: 零信任模式的账号仍在浏览器端加密
- **服务器盲**: Wisp 代理仍为盲转发（无法读取 TLS 内容）
- **双模式**: 用户可自由选择零信任或账号池模式

## 🎯 架构设计亮点

### 1. 双模式共存

- **零信任模式**: 客户端账号，完全隐私
- **账号池模式**: 服务器账号，共享便利
- **自由切换**: 用户根据需求选择

### 2. 智能账号调度

- **负载均衡**: 平均分配使用次数
- **健康监控**: 自动检测账号状态
- **故障转移**: 自动跳过不可用账号
- **限流保护**: 避免触发 Apple 限制

### 3. 安全与便利平衡

- **服务器加密**: AES-256-GCM 保护账号
- **管理员隔离**: API Key 认证分离权限
- **用户无感**: 一键下载，无需配置
