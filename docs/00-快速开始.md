# 第一部分：能力边界

## ✅ AssppWeb 可以做什么

### 1. 下载 App Store 上架的应用

**任何区域、任何语言的 App Store 上架应用**，只要账号有访问权限:

| 应用类型 | 是否支持 | 说明 |
|---------|---------|------|
| 免费应用 | ✅ 支持 | 如 Instagram、WhatsApp、TikTok |
| 付费应用（已购买） | ✅ 支持 | 账号必须已购买过该应用 |
| 内购应用 | ✅ 支持下载 | 应用本身可下载，内购功能取决于账号 |
| 订阅制应用 | ✅ 支持下载 | 如 Netflix、Spotify（下载后需账号激活） |

**示例**:
- 国区下架的应用（用美区账号下载 Instagram）
- 某区域独占的应用（日区游戏、美区工具）
- 国际版应用（TikTok 而非抖音）

### 2. 下载应用的历史版本

**前提条件**: 账号之前下载/购买过该应用

**工作原理**:
```
1. 账号已购买过应用 A
   ↓
2. 调用版本列表 API (getVersionListing)
   ↓
3. Apple 返回该账号可访问的历史版本列表
   ↓
4. 选择特定版本下载
```

**限制**:
- Apple 服务器必须保留该版本记录
- 账号必须在该版本发布时或之后购买过应用
- 部分应用的旧版本可能已被 Apple 删除

### 3. 跨区域访问应用

**多账号管理，不同区域独立**:

| 区域 | 账号 | 可下载应用示例 |
|------|------|--------------|
| 美国 (US) | us@example.com | Instagram, Twitter, ChatGPT |
| 中国 (CN) | cn@example.com | 微信、支付宝、抖音 |
| 日本 (JP) | jp@example.com | LINE、原神（日服）、モンスト |
| 欧洲 (GB/DE) | eu@example.com | 特定区域独占应用 |

## ❌ AssppWeb 不能做什么

### 1. 不能下载非 App Store 上架的应用

| 应用类型 | 是否支持 | 原因 |
|---------|---------|------|
| 内部测试应用 | ❌ 不支持 | 从未上架 App Store |
| TestFlight 测试版 | ❌ 不支持 | 独立的分发机制，需邀请码 |
| 企业签名应用 | ❌ 不支持 | 不通过 App Store 分发 |
| 个人开发未上架应用 | ❌ 不支持 | Apple API 无法访问 |
| 已彻底下架应用（未购买过） | ❌ 不支持 | Apple 服务器已删除 |

**为什么不支持?**

AssppWeb 的所有数据都来自 **Apple 官方 API**:

```
iTunes Search API → 搜索应用元数据
Apple Store API → 购买许可
Apple CDN → 下载 IPA 文件
```

如果 Apple 服务器上不存在该应用，AssppWeb 就无法获取。

### 2. 不能绕过 Apple 的限制

| 限制类型 | 说明 |
|---------|------|
| 区域锁定 | 如果应用仅在某区上架，必须使用该区账号 |
| 付费应用 | 必须先购买，不能免费获取付费应用 |
| 账号限制 | 被锁定/禁用的账号无法使用 |
| 设备兼容性 | 应用的系统要求（如 iOS 15+）无法修改 |

### 3. 不能修改应用内容

- 不能破解付费应用变免费
- 不能移除应用内广告
- 不能修改应用的内购项目
- 不能绕过应用的服务器验证

**AssppWeb 只是获取原版 IPA，内容完全由 Apple 提供，无任何修改。**

# 第二部分：使用指南

## 准备工作

### 你需要准备

1. **一个外区 Apple ID**
   - 例如：美区、日区、港区账号
   - 必须知道账号密码
   - 建议使用专门的下载账号，不要用主力账号

2. **AssppWeb 服务地址**
   - 自己部署的实例：如 `http://你的服务器IP:8080`
   - 或者朋友提供的地址

3. **目标应用的信息**
   - 应用名称，或者
   - Bundle ID（如 `com.example.app`）

---

## 第一步：添加 Apple ID 账号

### 1.1 访问 AssppWeb

打开浏览器，访问你的 AssppWeb 地址：

```
http://你的服务器IP:8080
```

你会看到主界面，左侧是导航栏。

### 1.2 进入账号页面

点击左侧导航栏的 **"Accounts"**（账号）。

### 1.3 添加新账号

1. 点击右上角的 **"Add Account"**（添加账号）按钮
2. 在弹出的表单中填写：

```
┌─────────────────────────────────────────┐
│  Add Apple Account                      │
├─────────────────────────────────────────┤
│                                         │
│  Email:  [your-id@icloud.com         ] │
│          ↑ 你的 Apple ID 邮箱           │
│                                         │
│  Password: [******************        ] │
│             ↑ 账号密码                  │
│                                         │
│  Country: [🇺🇸 United States         ▾] │
│           ↑ 账号所属地区（重要！）       │
│                                         │
│  Device ID: [a1b2c3d4e5f6            ] │
│             ↑ 设备标识符（可随机生成）   │
│             [🎲 Randomize]  ← 点击生成  │
│                                         │
│         [Cancel]  [Add Account]         │
└─────────────────────────────────────────┘
```

### 1.4 字段说明

#### Email（必填）
- 你的 Apple ID 邮箱
- 例如：`yourname@icloud.com`

#### Password（必填）
- 账号密码
- ⚠️ 密码存储在浏览器的 IndexedDB 中，服务器永远看不到

#### Country（必填）
- **非常重要！** 必须选择账号所属的真实地区
- 如果是美区账号，选择 `🇺🇸 United States`
- 如果是日区账号，选择 `🇯🇵 Japan`
- 选错会导致认证失败

#### Device ID（必填）
- 12 位十六进制字符，用于模拟设备标识
- 不知道填什么？点击 **"Randomize"** 按钮随机生成一个
- 每个账号的 Device ID 应该不同

---

## 第二步：搜索应用

### 2.1 进入搜索页面

点击左侧导航栏的 **"Search"**（搜索）。

### 2.2 搜索方式

AssppWeb 提供两种搜索方式：

#### 方式 A: 按应用名称搜索（推荐）

```
┌─────────────────────────────────────────┐
│  🔍 Search App Store                    │
├─────────────────────────────────────────┤
│                                         │
│  Search by:  ● Name    ○ Bundle ID      │
│                                         │
│  App Name: [Telegram                 ] │
│            ↑ 输入应用名称                │
│                                         │
│  Country:  [🇺🇸 United States        ▾] │
│            ↑ 选择搜索地区                │
│                                         │
│            [Search]                     │
└─────────────────────────────────────────┘
```

- 输入应用名称（如 `Telegram`）
- 选择地区（如 `🇺🇸 United States`）
- 点击 **"Search"**

#### 方式 B: 按 Bundle ID 搜索（精确）

```
┌─────────────────────────────────────────┐
│  🔍 Search App Store                    │
├─────────────────────────────────────────┤
│                                         │
│  Search by:  ○ Name    ● Bundle ID      │
│                                         │
│  Bundle ID: [ph.telegra.Telegraph    ] │
│             ↑ 输入 Bundle ID             │
│                                         │
│  Country:   [🇺🇸 United States       ▾] │
│                                         │
│            [Search]                     │
└─────────────────────────────────────────┘
```

- 输入完整的 Bundle ID
- 这种方式更精确，但需要提前知道 Bundle ID

## 第三步：获取许可

### 3.1 什么是"获取许可"？

在下载应用之前，你需要先"获取"（Purchase）该应用的许可证：

- 对于**免费应用**：这个操作相当于在 App Store 点击"获取"按钮
- Apple 会在你的账号下记录"你已获取过这个应用"
- 这个过程不需要付费

### 3.2 点击 "Get License"

在搜索结果中，点击应用卡片上的 **"Get License"** 按钮，选择对应地区的账号。

**重要提示**:
- 下载美区应用，选择美区账号
- 下载日区应用，选择日区账号
- ⚠️ 地区不匹配会失败

通常需要 5-10 秒，成功后会显示：

```
✅ License acquired successfully!
```

---

## 第四步：下载安装

### 4.1 创建下载任务

获取许可成功后，点击 **"Download"** 按钮，或者进入左侧导航栏的 **"Downloads"**（下载）页面。

### 4.2 下载状态说明

| 状态 | 说明 |
|-----|------|
| ⏳ **Downloading** | 正在从 Apple CDN 下载 IPA 文件 |
| 🔧 **Injecting** | 正在注入 SINF 许可证数据 |
| ✅ **Completed** | 下载完成，可以安装 |
| ❌ **Failed** | 下载失败（点击查看错误详情） |

### 4.3 安装方式

#### 方式 A: iPhone/iPad 直接安装（最简单）

1. 在 iOS 设备的 Safari 浏览器中访问 AssppWeb
2. 进入 Downloads 页面
3. 点击 **"Install"** 按钮
4. iOS 会弹出安装确认对话框，点击 **"Install"**
5. 应用会开始安装到主屏幕

#### 方式 B: 使用第三方工具（AltStore/Sideloadly）

1. 点击 **"Copy URL"** 复制 IPA 下载链接
2. 使用电脑上的 AltStore 或 Sideloadly：
   - 打开工具
   - 选择"从 URL 安装"
   - 粘贴刚才复制的链接
   - 按照工具的提示完成安装

#### 方式 C: 下载 IPA 文件

1. 点击 **"Copy URL"** 复制链接
2. 在浏览器中访问这个链接
3. IPA 文件会下载到你的电脑
4. 使用任何 iOS 侧载工具（如 Sideloadly、iMazing）安装

### 4.4 安装后验证

安装完成后，首次打开时，iOS 可能会提示需要信任开发者：

1. 点击 **"Settings"**，进入：
   ```
   设置 > 通用 > VPN与设备管理 > 开发者应用
   ```

2. 找到对应的证书，点击 **"信任"**

3. 返回主屏幕，再次打开应用即可

# 第三部分：常见问题

## Q1: 可以下载国区下架的应用吗？

**A**: ✅ 可以，前提是该应用在其他区域仍然上架。

**示例**:
- Instagram 国区下架 → 用美区账号下载 ✅
- Telegram 国区下架 → 用美区/港区账号下载 ✅
- TikTok 国区下架 → 用美区账号下载 ✅

**操作步骤**:
1. 准备一个外区 Apple ID（如美区）
2. 在 AssppWeb 添加该账号
3. 搜索应用并选择对应区域
4. 系统自动使用外区账号下载

---

## Q2: 可以下载已经彻底下架的应用吗（如 Flappy Bird）？

**A**: ⚠️ 取决于账号购买历史

**情况 1: 账号之前购买过该应用**
- ✅ 可以重新下载
- 从账号的「已购项目」中获取
- 即使应用已下架，Apple 仍保留已购记录

**情况 2: 账号从未购买过该应用**
- ❌ 无法下载
- Apple Search API 不会返回已下架应用
- 无法通过 AssppWeb 获取

---

## Q3: 可以下载应用的旧版本吗？

**A**: ⚠️ 有条件支持

**前提条件**:
1. 账号之前下载/购买过该应用
2. Apple 服务器保留了旧版本记录
3. 账号在该版本发布后购买过应用

**操作流程**:
1. 在搜索结果页面，点击 **"View Details"**
2. 点击 **"Version History"** 标签
3. 选择你想下载的旧版本
4. 点击 **"Get License"** 获取该版本的许可

**常见场景**:
- ✅ 微信 6.5.8（账号之前下载过微信）
- ✅ 抖音 18.0.0（回退到旧版本）
- ❌ 从未下载过的应用的旧版本（无购买记录）

---

## Q4: 需要付费的应用可以免费下载吗？

**A**: ❌ 不可以

**原理**:
- AssppWeb 调用的是 Apple 官方购买 API
- 付费应用需要账号先完成付费
- 无法绕过 Apple 的支付系统

**但是**:
- ✅ 如果账号已购买过该应用，可以重新下载 IPA
- ✅ 免费应用可以直接下载（获取许可免费）

---

## Q5: TestFlight 的测试版可以下载吗？

**A**: ❌ 不可以

**原因**:
- TestFlight 使用独立的分发系统
- 不通过 App Store API 分发
- 需要开发者邀请和专用 TestFlight 客户端

**替代方案**:
- 使用 TestFlight 官方客户端
- 开发者提供 IPA 文件直接安装

---

## Q6: 企业签名的应用可以下载吗？

**A**: ❌ 不可以

**原因**:
- 企业签名应用通过 Apple Enterprise Program 分发
- 不经过 App Store
- AssppWeb 只能访问 App Store API

---

## Q7: 账号会不会被封？

**A**: ⚠️ 极低风险，但需注意使用频率

**AssppWeb 的安全设计**:
- ✅ 零信任架构：服务器看不到凭据
- ✅ 使用官方 API：与 Apple Configurator 相同
- ✅ 盲代理模式：行为与正常下载一致

**可能触发风控的行为**:
- ❌ 短时间内大量下载（如1小时内下载50个应用）
- ❌ 多个设备同时使用同一账号
- ❌ 频繁切换 IP 地址（使用代理）

**建议的安全实践**:
- ✅ 设置频率限制（每账号每小时最多10次）
- ✅ 账号使用后冷却5-10分钟
- ✅ 使用稳定的 IP 地址
- ✅ 避免多设备并发登录

---

## Q8: 为什么有些应用搜不到？

**可能原因**:

1. **区域选择错误**
   - 应用仅在特定区域上架
   - 解决: 切换到正确的区域搜索

2. **Bundle ID 输入错误**
   - 拼写错误或格式不对
   - 解决: 检查 Bundle ID（如 `com.instagram.ios`）

3. **应用已彻底下架**
   - Apple 已从所有区域移除
   - 解决: 无法通过 AssppWeb 获取

4. **iTunes API 延迟**
   - 新上架的应用可能需要几小时同步
   - 解决: 稍后重试

---

## 故障排查

### 添加账号失败，提示"Authentication failed"

**可能原因**：

1. **密码错误**
   - 解决：检查密码拼写
   - 如果账号开启了 2FA（双重认证），需要使用"应用专用密码"

2. **地区选择错误**
   - 解决：确认选择的 Country 与账号实际地区一致

3. **账号被锁定**
   - 解决：访问 appleid.apple.com 解锁账号

4. **服务器网络问题**
   - 解决：检查服务器能否访问 `*.apple.com`

---

### 搜索不到应用

**可能原因**：

1. **应用在该地区未上架**
   - 解决：换一个地区试试（如美区换成日区）

2. **应用已下架**
   - 解决：AssppWeb 无法下载已下架的应用

3. **名称拼写错误**
   - 解决：检查拼写，或使用 Bundle ID 搜索

4. **应用是付费的**
   - 解决：AssppWeb 只支持免费应用

---

### 获取许可失败，提示"Region mismatch"

**原因**：账号地区与应用地区不匹配

**解决**：
- 确保使用对应地区的账号
- 例如：下载美区应用必须用美区账号

---

### 下载卡在 0% 或中途失败

**可能原因**：

1. **服务器网络不稳定**
   - 解决：刷新页面，重新下载

2. **Apple CDN 限速**
   - 解决：等待一段时间后重试

3. **应用体积过大，超时**
   - 解决：增加服务器带宽，或使用 Cloudflare 加速

---

### 安装到 iPhone 时提示"无法验证应用"

**原因**：iOS 需要信任开发者证书

**解决**：
1. 进入：设置 > 通用 > VPN与设备管理
2. 找到对应证书，点击"信任"

---

### 应用打开后闪退

**可能原因**：

1. **iOS 版本太低**
   - 解决：检查应用的最低系统要求，更新 iOS

2. **应用需要登录，但账号地区不对**
   - 解决：部分应用（如 YouTube）需要登录对应地区的账号

3. **SINF 注入失败**
   - 解决：重新下载 IPA

---

## 使用技巧

### ✅ 最佳实践

1. **使用专门的下载账号**
   - 不要用你的主力 Apple ID
   - 避免影响你的真实账号

2. **为每个账号生成不同的 Device ID**
   - 点击"Randomize"按钮随机生成
   - 避免多个账号使用相同的设备标识

3. **定期检查账号状态**
   - 如果出现"Authentication failed"，重新添加账号
   - Apple 的登录 token 会过期

4. **下载完成后及时清理**
   - AssppWeb 不会自动删除 IPA 文件
   - 下载完成后，点击"Delete"清理服务器空间

5. **使用 Cloudflare 加速**
   - 如果下载速度慢，配置 Cloudflare CDN
   - 参考：[02-部署运维指南](./02-部署运维指南.md)

---

### ⚠️ 常见错误习惯

1. ❌ 使用主力 Apple ID
   - 可能导致账号被锁定或异常

2. ❌ 地区选择错误
   - 美区账号选择了日本，导致认证失败

3. ❌ 多个账号使用相同 Device ID
   - 可能被 Apple 检测为异常行为

4. ❌ 下载付费应用
   - AssppWeb 只支持免费应用，付费应用会失败

5. ❌ 期望下载已下架的应用
   - App Store 搜索不到的应用，AssppWeb 也无法下载

## 总结

### AssppWeb 的核心价值

1. **零信任安全**: 服务器永远看不到你的 Apple 凭据
2. **跨区域访问**: 轻松获取外区应用，无需复杂配置
3. **版本管理**: 保留和回退到特定版本
4. **批量管理**: 账号池模式适合团队和机构

### 核心限制

1. **仅限 App Store 上架应用**: 无法获取非上架应用
2. **需要对应区域账号**: 无法绕过 Apple 的区域限制
3. **付费应用需先购买**: 无法免费获取付费应用

### 适用人群

- ✅ 需要下载外区应用的个人用户
- ✅ 管理多个 Apple ID 的开发团队
- ✅ 为学生/员工批量提供应用的机构
- ✅ 需要应用版本管理的测试人员

### 不适用人群

- ❌ 需要下载破解/修改应用的用户
- ❌ 需要 TestFlight 测试版的开发者
- ❌ 需要企业内部应用分发的公司
