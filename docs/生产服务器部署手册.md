## 📋 部署信息

- **服务器**: 156.226.173.202

## 🚀 手动部署步骤

### 步骤 1: SSH 登录服务器

```bash
ssh root@156.226.173.202
```

### 步骤 2: 进入项目目录

```bash
cd /root/AssppWeb
```

### 步骤 3: 拉取最新代码

```bash
git pull origin main
```

### 步骤 4: 检查是否有现有配置

```bash
# 查看是否有 .env.generated
ls -la .env.generated

# 如果存在，查看现有的 API Key
cat .env.generated
```

### 步骤 5: 停止旧服务

```bash
docker compose down
```

### 步骤 6: 构建新镜像

```bash
docker compose build
```

### 步骤 7: 启动服务

**如果有现有配置（推荐）**:
```bash
# 加载现有配置
source .env.generated

# 启动服务
export ADMIN_API_KEY
export ACCOUNT_POOL_KEY
docker compose up -d
```

**如果是首次部署**:
```bash
# 运行部署脚本（会自动生成密钥）
bash deploy-manual.sh
```

### 步骤 8: 检查服务状态

```bash
# 查看容器状态
docker compose ps

# 查看日志
docker compose logs --tail=50

# 查看实时日志
docker compose logs -f
```

### 步骤 9: 测试服务

```bash
# 测试 API
curl -s https://chatvpn.site/api/settings | jq .

# 测试管理员 API（使用你的 API Key）
curl -s -H "x-admin-key: YOUR_API_KEY" https://chatvpn.site/api/admin/pool/stats | jq .
```

## 🔑 密钥管理

### 查看现有密钥

```bash
cat .env.generated
```

输出示例：
```
ADMIN_API_KEY=04ecc4cecd78b29e09af22dc62a473dd59a6413ffbef9f6f4ec4f83e495d2c64
ACCOUNT_POOL_KEY=f15f0de35176cd5da24d5950fbc111c67e77d60b333adf767a563618d9b2688f
```

⚠️ **重要提示**:
- 请妥善保存这些密钥！
- `ACCOUNT_POOL_KEY` 丢失后，所有账号密码将无法解密！

### 如果需要重新生成密钥

```bash
# ⚠️ 警告: 这会清空数据库！
rm -f .env.generated
rm -rf mnt/asspp-data/*
bash deploy-manual.sh
```

## 📋 部署后配置

### 1. 配置管理员

访问: https://chatvpn.site

1. 点击 **"管理"** → **"管理员设置"**
2. 输入服务器的 `ADMIN_API_KEY`
3. 点击 **"测试连接"** 和 **"保存配置"**

### 2. 添加账号到账号池

**方式 A - Web 界面**:
- 管理 → 账号池管理 → 添加账号

**方式 B - CLI 工具**:
```bash
bash admin-cli.sh pool:add
```

### 3. 添加白名单应用

**方式 A - Web 界面**:
- 管理 → 白名单管理 → 搜索应用 → 添加

**方式 B - CLI 工具**:
```bash
bash admin-cli.sh whitelist:add
```

## 🔍 故障排查

### 服务无法启动

```bash
# 查看详细日志
docker compose logs --tail=100

# 检查容器状态
docker compose ps

# 重启服务
docker compose restart
```

### 环境变量未生效

```bash
# 确保已加载环境变量
source .env.generated
export ADMIN_API_KEY
export ACCOUNT_POOL_KEY

# 重启服务
docker compose down
docker compose up -d
```

### 数据库问题

```bash
# 查看数据库文件
ls -la mnt/asspp-data/

# 如果需要重置（⚠️ 会清空所有数据）
docker compose down
rm -rf mnt/asspp-data/*
docker compose up -d
```
