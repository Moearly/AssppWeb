## 部署前准备

### 系统要求

- Docker 和 Docker Compose
- 服务器至少 1GB 内存
- 端口 8080 可用（或自定义）

### 环境变量

部署脚本会自动生成以下密钥：

```bash
# 管理员 API Key（64字符十六进制）
ADMIN_API_KEY=<自动生成>

# 账号池加密密钥（64字符十六进制）
ACCOUNT_POOL_KEY=<自动生成>
```

## 快速部署

### 方式一：自动部署（推荐）

执行部署脚本，自动生成密钥并启动服务：

```bash
bash deploy-manual.sh
```

脚本会自动：
1. ✅ 生成管理员 API Key
2. ✅ 生成账号池加密密钥
3. ✅ 保存配置到 `.env.generated`
4. ✅ 初始化数据库
5. ✅ 构建 Docker 镜像
6. ✅ 启动服务

部署完成后，脚本会显示：

```
📋 管理员配置
====================

🔑 管理员 API Key:
   <你的API Key>

📱 使用方法:
   1. 打开浏览器访问: http://localhost:8080
   2. 点击 "管理" → "管理员设置"
   3. 输入上面的 API Key
   4. 点击 "测试连接" 和 "保存配置"
```

### 方式二：手动部署

1. **生成密钥**

```bash
# 生成管理员 API Key
export ADMIN_API_KEY=$(openssl rand -hex 32)
echo "ADMIN_API_KEY=$ADMIN_API_KEY"

# 生成账号池加密密钥
export ACCOUNT_POOL_KEY=$(openssl rand -hex 32)
echo "ACCOUNT_POOL_KEY=$ACCOUNT_POOL_KEY"
```

2. **保存密钥到文件**

```bash
cat > .env.generated << EOF
ADMIN_API_KEY=$ADMIN_API_KEY
ACCOUNT_POOL_KEY=$ACCOUNT_POOL_KEY
EOF
```

3. **构建并启动服务**

```bash
docker compose build
docker compose up -d
```

4. **检查服务状态**

```bash
docker compose ps
docker compose logs -f
```

## 管理员配置

### 1. 访问管理后台

打开浏览器，访问：

```
http://your-server:8080
```

或者生产环境：

```
https://your-domain.com
```

### 2. 配置 API Key

1. 点击左侧导航 **"管理"** → **"管理员设置"**
2. 在 **"API Key"** 输入框中输入部署时生成的 `ADMIN_API_KEY`
3. 点击 **"测试连接"** 验证密钥是否有效
4. 如果测试成功，点击 **"保存配置"**

![管理员设置页面示例](../assets/admin-settings.png)

### 3. 权限说明

配置成功后，管理员可以访问以下功能：

- ✅ **账号池管理** - 添加、删除、启用/禁用 Apple ID 账号
- ✅ **白名单管理** - 添加、编辑、删除可下载的应用
- ✅ **健康检查** - 检查账号有效性，查看健康日志
- ✅ **统计信息** - 查看账号池使用情况和统计数据

## 账号池管理

### 添加账号

1. 进入 **"管理"** → **"账号池管理"**
2. 点击右上角 **"添加账号"**
3. 填写账号信息：
   - **邮箱地址** - Apple ID 邮箱（必填）
   - **密码** - Apple ID 密码（必填）
   - **国家/地区** - 账号所属区域（默认 US）
   - **设备标识符** - 自动生成或手动填写

4. 点击 **"添加"**

⚠️ **重要提示**：
- 密码使用 AES-256-GCM 加密存储
- 添加账号后会自动进行认证测试
- 如需二次验证（2FA），会弹出输入框

### 批量导入账号

1. 点击 **"批量导入"**
2. 按以下格式输入账号信息（每行一个）：

```
email1@example.com:password1
email2@example.com:password2
email3@example.com:password3
```

3. 选择国家/地区
4. 点击 **"导入"**

### 管理账号状态

- **启用/禁用** - 点击账号卡片的 **"启用"** / **"禁用"** 按钮
- **删除账号** - 点击 **"删除"** 按钮（需确认）
- **健康检查** - 点击 **"检查健康"** 测试账号是否可用

### 账号状态说明

| 状态 | 说明 | 操作 |
|------|------|------|
| 🟢 `active` | 活跃可用 | 正常分配给用户使用 |
| 🔴 `disabled` | 手动禁用 | 不会分配给用户 |
| ⚠️ `expired` | 令牌过期 | 需要重新认证 |
| ⏳ `in_use` | 使用中 | 已分配给用户，等待释放 |

- **导出白名单** - 导出为 JSON 格式备份

## 常见问题

### 1. 忘记 API Key 怎么办？

**解决方案：**

```bash
# 在服务器上查看配置文件
cat .env.generated | grep ADMIN_API_KEY

# 或者重新生成新的 API Key
export ADMIN_API_KEY=$(openssl rand -hex 32)
echo "新的 ADMIN_API_KEY: $ADMIN_API_KEY"

# 重启服务使新密钥生效
docker compose restart
```

### 2. API Key 泄露了怎么办？

**解决方案：**

```bash
# 1. 生成新的 API Key
export ADMIN_API_KEY=$(openssl rand -hex 32)

# 2. 更新配置文件
echo "ADMIN_API_KEY=$ADMIN_API_KEY" > .env.generated

# 3. 重启服务
docker compose restart

# 4. 通知所有管理员更新 API Key
```

### 3. 为什么提示 "Unauthorized"？

**可能原因：**
- API Key 输入错误
- 服务器未配置 `ADMIN_API_KEY` 环境变量
- 浏览器缓存了旧的 API Key

**解决方案：**
1. 检查 API Key 是否正确（注意空格）
2. 重启浏览器清除缓存
3. 在服务器上检查环境变量：
   ```bash
   docker compose exec asspp printenv | grep ADMIN_API_KEY
   ```

### 4. 账号添加失败 "invalid credentials"？

**可能原因：**
- Apple ID 密码错误
- 账号需要二次验证（2FA）
- 账号被锁定或需要重置密码

**解决方案：**
1. 先在真实 iPhone 上登录该 Apple ID，确保账号可用
2. 如有 2FA，准备好验证码
3. 确认账号没有任何安全警告

### 5. 账号池加密密钥丢失怎么办？

**⚠️ 严重问题！**

如果 `ACCOUNT_POOL_KEY` 丢失，所有已存储的账号密码将**无法解密**。

**解决方案：**
1. 删除旧数据库：
   ```bash
   rm data/pool.db
   ```
2. 重新生成加密密钥
3. 重新添加所有账号

**预防措施：**
- 定期备份 `.env.generated` 文件
- 将加密密钥存储到安全的密钥管理系统
- 使用 `git-crypt` 或 `SOPS` 加密存储敏感配置

### 6. 如何备份账号池数据？

**备份方案：**

```bash
# 1. 备份数据库
docker compose exec asspp cp /app/data/pool.db /app/data/pool.db.backup
docker cp assppweb-asspp-1:/app/data/pool.db.backup ./backup/

# 2. 备份配置文件
cp .env.generated ./backup/.env.generated.backup

# 3. 打包备份
tar -czf backup-$(date +%Y%m%d).tar.gz backup/
```

**恢复方案：**

```bash
# 1. 解压备份
tar -xzf backup-20260310.tar.gz

# 2. 恢复配置
cp backup/.env.generated.backup .env.generated

# 3. 恢复数据库
docker cp backup/pool.db.backup assppweb-asspp-1:/app/data/pool.db

# 4. 重启服务
docker compose restart
```

### 7. 如何迁移到新服务器？

**迁移步骤：**

1. **在旧服务器上备份**：
   ```bash
   # 备份数据库和配置
   docker compose down
   tar -czf assppweb-backup.tar.gz data/ .env.generated
   ```

2. **传输到新服务器**：
   ```bash
   scp assppweb-backup.tar.gz new-server:/root/
   ```

3. **在新服务器上恢复**：
   ```bash
   # 解压备份
   tar -xzf assppweb-backup.tar.gz
   
   # 克隆代码
   git clone https://github.com/your-repo/AssppWeb.git
   cd AssppWeb
   
   # 恢复数据和配置
   cp ../data/ ./data/
   cp ../.env.generated ./.env.generated
   
   # 加载环境变量并启动
   source .env.generated
   docker compose up -d --build
   ```

## 安全建议

### 1. 生产环境部署

- ✅ 使用 HTTPS（通过 Nginx/Caddy 反向代理）
- ✅ 设置防火墙规则，限制管理端口访问
- ✅ 使用强密码生成器生成 API Key（64字符+）
- ✅ 定期轮换 API Key（每3个月）
- ✅ 启用访问日志审计

### 2. API Key 管理

- ❌ 不要在代码中硬编码 API Key
- ❌ 不要将 `.env.generated` 提交到 Git
- ✅ 使用环境变量或密钥管理服务
- ✅ 限制 API Key 的访问权限
- ✅ 为不同管理员分配不同的 API Key（未来功能）

### 3. 数据保护

- ✅ 定期备份数据库（每天/每周）
- ✅ 使用加密存储敏感配置
- ✅ 限制数据库文件权限（chmod 600）
- ✅ 监控异常登录行为

### 查看服务日志

```bash
# 实时日志
docker compose logs -f

# 仅查看最近100行
docker compose logs --tail=100

# 搜索特定关键词
docker compose logs | grep "ERROR"
```

### 健康检查

```bash
# 检查服务状态
docker compose ps

# 检查服务健康
curl http://localhost:8080/api/settings

# 检查管理员权限
curl -H "x-admin-key: YOUR_API_KEY" http://localhost:8080/api/admin/pool/stats
```

建议使用 Prometheus + Grafana 监控：
- 账号池使用率
